1. 项目概述
本工具面向 Windows 平台，是可视化图形程序，实现图形界面选择 YOLO 版本、全自动搭建 Conda 深度学习环境，用户无需手动输入任何终端命令。
支持部署版本：YOLOv5、YOLOv7、YOLOv9 原版、YOLOv10（清华原版）、YOLOv8、YOLO11
2. 技术选型
开发语言：Python
GUI 框架：PyQt6
配置解析：pyyaml
打包工具：PyInstaller（输出独立 exe）
3. 整体架构与调用流程
plaintext

    ↓
功能核心模块
├─环境扫描模块 env_scan
├─Conda环境处理模块 conda_handler
├─YOLO安装调度模块 yolo_installer

5. 各模块详细功能说明
5.1 env_scan.py｜环境扫描模块
模块定位：程序初始化最先运行，探测本机基础依赖，为后续安装提供判断依据。
功能清单
自动遍历系统常见路径，查找conda.exe，验证 conda 程序有效性，返回完整绝对路径；未找到返回空值。
执行 git --version，检测 Git 是否安装并配置系统环境变量。
执行 nvidia-smi，检测是否存在 NVIDIA 独立显卡，区分 GPU/CPU 部署场景。
封装通用终端执行工具函数，统一捕获控制台输出、返回码、异常信息。
对外输出
conda 路径、Git 可用状态、显卡存在状态、完整扫描日志
约束
独立模块，不依赖其他自定义代码，支持单独调试运行。
5.2 conda_handler.py｜Conda 处理模块
模块定位：统一封装所有 conda、pip 相关操作，解决 Windows 无法直接执行conda activate的经典问题；上层程序无需手写零散终端命令。
功能清单
创建指定名称、指定 Python 版本虚拟环境，附带-y参数自动确认交互。
核心函数 run_in_env()：使用conda run -n 环境名语法，在隔离虚拟环境内部执行 pip/python 指令。
封装两套 PyTorch 安装接口
GPU 版：CUDA12.1 配套 torch、torchvision、torchaudio
CPU 版：通用无 CUDA 版本 Pytorch
底层封装进程调用，统一收集 stdout 标准输出、stderr 错误日志，返回执行状态。
预留扩展接口：删除虚拟环境、查询本地已存在环境列表。
入参
初始化必须传入有效 conda.exe 绝对路径
对外开放方法
create_env()、run_in_env()、install_torch_cuda121()、install_torch_cpu()
5.3 yolo_installer.py｜YOLO 安装调度模块
模块定位：软件业务核心，读取配置区分两大安装路线，自动化执行整套部署流水线。
功能清单
启动加载repos.yaml配置文件，读取全部 YOLO 版本信息。
自动区分两种部署模式
source 源码模式：YOLOv5 / YOLOv7 / YOLOv9 原版 / YOLOv10 清华
执行流程：安装 Pytorch → 判断本地项目目录是否存在 → git clone 拉取源码 → pip install -r requirements.txt
pip 库模式：YOLOv8 / YOLO11
执行流程：安装 Pytorch → pip install ultralytics
采用分步生成器机制，每完成一个步骤，实时向外推送步骤名称、运行日志、执行状态；任意步骤失败立即终止整条流水线。
统一管理源码存放目录yolo_workspace，自动创建文件夹，规避路径不存在报错。
入参
Conda 处理实例对象、虚拟环境名称、选中 YOLO 版本、是否启用 GPU
依赖模块
conda_handler
5.4 auto_test.py｜自动化测试模块（可选模块）
模块定位：部署结束后自动验证环境可用性，确认 YOLO 能够正常完成推理，避免表面安装成功实际运行报错。
功能清单
根据安装模式区分测试命令
源码版本：运行项目内 detect.py 执行图片推理
ultralytics pip 版本：执行简短 Python 代码加载模型测试
支持加载默认测试图片，捕获程序运行完整日志。
识别报错信息，区分环境正常、模块缺失、CUDA 异常等故障类型。
将验证结果推送至日志面板展示。
触发时机
YOLO 整套安装流程全部执行完毕后选择性调用。
5.5 main.py｜UI 界面主模块
模块定位：人机交互入口，管理窗口控件、接收用户操作、多线程调度，串联全部底层模块。
强制规范：所有耗时安装任务必须放入独立 QThread 子线程，禁止主线程执行命令，防止窗口卡死。
功能清单
界面控件管理
状态标签：展示 Conda 扫描结果
【重新扫描环境】按钮，手动触发环境检测
下拉选择框：选择目标 YOLO 版本
复选框：切换 GPU/CPU 安装模式
【开始一键安装部署】启动按钮
只读日志文本框，实时输出运行信息
Qt 信号槽通信：子线程自定义信号持续向主线程推送日志、任务结束通知。
前置校验拦截：未检测到 Conda 时直接拦截安装任务，弹窗提示用户。
任务锁机制：安装运行期间锁定全部按钮，禁止重复启动部署任务。
任务结束回调：安装完成 / 异常终止后解除控件锁定，展示最终运行结果。
依赖
PyQt6、所有自定义功能模块
5.6 repos.yaml｜静态配置文件
模块定位：集中管理 YOLO 版本参数；后期新增 YOLO 版本仅修改配置文件，无需改动业务代码。
存储字段
YOLO 界面展示名称
mode：source（源码部署） /pip（库安装）
git_url：源码仓库地址（source 模式生效）
pkg_name：pip 包名称（pip 模式生效）
默认 Python 版本、默认虚拟环境名称
6. 完整开发实施步骤
阶段 1：前期环境准备
本地安装 Python、Miniconda3、Git
预先安装依赖库：PyQt6、pyyaml
阶段 2：模块分步开发与单元测试（按顺序开发）
开发env_scan.py，独立脚本测试，验证可正常识别 conda、git、显卡。
开发conda_handler.py，单独测试创建虚拟环境、环境内执行 pip 命令。
开发yolo_installer.py，编写配置文件，分别测试源码模式、pip 模式两条安装流程。
搭建基础 GUI 窗口，完成页面布局、按钮绑定（暂时不接入真实安装逻辑，仅打印测试日志）。
开发后台安装子线程，实现模块与 UI 对接，完成日志实时回显。
开发auto_test.py，接入安装流程末尾。
整体完整联调，逐个 YOLO 版本完整测试全流程。
阶段 3：打包发布
编写 build.bat 打包脚本，使用 PyInstaller 打包为独立 exe 程序
配置参数：关闭控制台窗口、单文件打包模式
测试打包后的 exe 能否正常运行
7. 核心难点与解决方案
GUI 执行安装时窗口无响应
解决方案：所有命令执行逻辑放置 QThread 子线程，主线程只负责界面刷新。
Windows 终端无法激活 conda 环境
解决方案：全程统一使用 conda run -n 环境名 语法，摒弃 activate 命令。
GitHub 克隆缓慢、超时失败
预留拓展：配置文件新增镜像地址字段，支持切换 Gitee 镜像仓库。
exe 打包后读取 yaml 配置文件失败
解决方案：统一使用相对路径，发布程序时同步附带 repos.yaml。
程序异常直接闪退
解决方案：所有命令执行增加 try 异常捕获，错误信息输出至日志面板。
8. 软件运行前置条件（交付附带说明）
操作系统：Windows
电脑预先安装 Miniconda3
预先安装 Git，并配置至系统环境变量
9. 成品操作流程
打开程序，点击【重新扫描环境】
在下拉框选择需要部署的 YOLO 版本
根据硬件情况勾选「使用 CUDA GPU 版本」
点击【开始一键安装部署】
持续观察日志窗口，等待自动执行；出现成功提示即部署完成
